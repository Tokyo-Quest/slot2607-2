# MID POP ワンボタン・パニック! prototype2

3段階ステージ構成版。**スロット → ミニゲーム → 裏ステージ** と勝ち進むほど上位の賞品になります。ボーナスステージ・復活チャレンジは廃止。

## 起動方法

`prototype2/index.html` をダブルクリック(Chrome / Edge 推奨)→ **F11** で全画面。完全オフライン動作(フォントのみ初回オンライン推奨、README下部参照)。

## ゲームフロー

1. **待機画面** → ボタンでスタート(1プレイ=**5回チャレンジ**)
2. **スロット**: 3回押して停止。**3つ揃い(5%)でミニゲームへ**。2つ揃いで「リーチ!!」表示
   - 揃わなければそのチャレンジは4等。クレジットが残っていれば再スロット
   - 画面隅に残チャレンジカウンター表示。ミニゲーム・裏ステージへ進んでも残クレジットは消えない
3. **ミニゲーム開始告知**(約5秒): メンバー名+ゲーム名+ルール説明
4. **ミニゲーム**(成功率目安30%): 失敗=3等
5. **裏ステージ突入演出** → **裏ステージ**(symbol_1〜5のスロット・my-slot-app素材)
   - ハズレ(70%)=2等 / 通常目そろい(30%)=1等 / 特別目 symbol_1 そろい(0%)=特賞(当選しない)
6. 全クレジット消費後、**そのプレイで最も良かった等級**で最終結果表示

## 操作・隠しキー(待機画面中)

| キー | 動作 |
|---|---|
| スペース / Enter / クリック | ゲームボタン |
| **1〜4** | 1等〜4等を強制(そのプレイの最初のチェーンに適用) |
| **6〜9** | ミニゲーム強制(6=明媚 7=葵 8=蕎麦 9=溺惑) |

※特賞の強制キーは操作ミス防止のため**存在しません**(特賞は絶対に出ない設計)
| **0** | 強制リセット |
| **D** | デバッグモード切替 |

## 確率調整(config.js)

- `play.credits` … 1プレイのチャレンジ回数(既定5)
- `slot.winRate` … スロット3つ揃い率(既定0.05)
- `slot.reachTeaseRate` … ハズレ時にリーチを見せる率
- `minigames.*` … 各ゲームの判定幅(成功率目安30%に調整済み。実力系は要実測)
- `ura.winRate` … 裏ステージ通常目そろい率(既定0.30)
- `ura.spWinRate` … 特別目そろい率(既定0=当選しない)
- `tiers` … 等級名・賞品名

## 裏ステージについて

`my-slot-app` の素材(bg_main / frame_reel / symbol_1〜5 / text_win等)と実音源(se_*.mp3 / bgm_spin.mp3)を `assets/images/ura/`・`assets/sounds/ura/` にコピーして使用。

**突入演出は現在CSSの仮演出**(暗転→ストロボ→「裏ステージ」)です。専用素材を作る場合は以下を推奨:

| ファイル名 | サイズ | 用途 |
|---|---|---|
| `ura_warning.png` | 1920×1080 透過 | 突入演出のロゴ/警告ビジュアル(`config.images.uraWarning` に設定) |
| `bgm_ura_intro.mp3` | - | 突入演出用SE/ジングル |

## prototype との違い

- ボーナス「4人まとめてキメろ!」・復活チャレンジを削除
- ミニゲーム進出条件: リーチ → **3つ揃い**(リーチ表示は演出として残存)
- カットインにルール説明を追加し約5秒表示
- 裏ステージ(第3段階)を追加
- 5クレジット制+隅カウンター
- 賞品体系: 特賞/1等/2等/3等/4等(到達ステージで決定)
